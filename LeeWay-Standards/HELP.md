# 📖 LEEWAY™ Terminal Command Encyclopedia

Welcome to the Sovereign CLI. This document provides a step-by-step guide on how to utilize the LEEWAY SDK to govern your codebase.

---

## 🚀 Step-by-Step Success Path

1.  **Initialize**: Run `leeway start` to meet Agent Lee and calibrate your system.
2.  **Diagnose**: Run `leeway doctor` to see if your environment meets the Sovereign Standards.
3.  **Audit**: Run `leeway audit` to see your current compliance score.
4.  **Align**: Run `leeway align --apply` to automatically fix any structural issues.
5.  **Scan**: Run `leeway scan` to ensure no secrets are leaked in your project.

## 🛠️ Command Reference

### `leeway start`
*   **WHAT**: Boot the system and enter the interactive Agent Lee console.
*   **WHEN**: Use this as your primary daily interface for project orchestration.
*   **WHY**: To interact with Lee directly, run complex tasks, and manage the Hive Mind.

### `leeway doctor`
*   **WHAT**: Comprehensive system health and compliance diagnosis.
*   **WHEN**: Run after first installation or when the system feels "clogged."
*   **WHY**: Ensures all paths, permissions, and dependencies are correctly tuned.

### `leeway audit`
*   **WHAT**: Scans the entire codebase and generates a LEEWAY Compliance Score.
*   **WHEN**: Run before major releases or during PR reviews.
*   **WHY**: To provide a deterministic metric of code quality and structural integrity.

### `leeway align`
*   **WHAT**: Detects files missing the mandatory 5W + H headers and injects them.
*   **WHEN**: Run whenever you add new files to the project.
*   **WHY**: To enforce identity and traceability across the entire stack. Use `--apply` to write changes.

### `leeway scan`
*   **WHAT**: Security scan for hardcoded secrets, keys, and tokens.
*   **WHEN**: Run before every `git commit`.
*   **WHY**: To prevent devastating security leaks in public or private repositories.

### `leeway registry`
*   **WHAT**: Rebuilds the `.leeway/registry.json` file.
*   **WHEN**: Run after large refactors or adding many new directories.
*   **WHY**: Updates the "Hive Mind's Map" so agents know where everything is.

### `leeway map`
*   **WHAT**: Generates a high-fidelity ASCII architecture map.
*   **WHEN**: Use for documentation or to explain the project structure to new devs.
*   **WHY**: Provides a visual representation of how the 7 Families are distributed.

### `leeway forge <id> <role>`
*   **WHAT**: Generates a new custom NPC agent file.
*   **WHEN**: When you need a specialized "guardian" for a specific part of your app.
*   **WHY**: To expand the capabilities of your sovereign system beyond the core 21 agents.

### `leeway awaken`
*   **WHAT**: Grants Agent Lee the power to utilize **LOCAL** Transformers and LLMs as tools.
*   **WHEN**: When you need to transport heavy logic between 'Local Galaxies' or perform complex analysis.
*   **WHY**: To expand the cognitive reach of the Hive Mind while ensuring 100% data privacy. This command also triggers a **Deep Learning Loop** where Lee studies all past successes and failures.

### `leeway wisdom`
*   **WHAT**: Retrieves the cumulative knowledge, levels, and insights of the Hive Mind.
*   **WHEN**: Whenever you want to see the evolution of your project's governance.
*   **WHY**: To maintain absolute transparency and trust between the Architect (you) and the Emissary (Lee).
